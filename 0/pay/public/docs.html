FazzPay API Reference

Automate premium 1080x1920 high-fidelity payment verification images for banking, e-wallet, or subscription flows. Integrates securely with your CRM or serverless stacks.

Private Authorization Guard

The security parameter key fazzganzz is private. Never leak this variable inside frontend production client scripts.

Endpoint Definition

POST /api/generate

Accepts application/json payloads and replies directly with high-definition, anti-aliased binary PNG payload stream headers.

Payload Parameters

Field Type Required Description
apiKey string True Security validation token ("fazzganzz").
amount string True Formatted currency string (e.g. "Rp 250.000").
productName string True Name of item, server subscription, or service.
buyerName string True First/Last name representation of the sender.
buyerUsername string False Target buyer @handle. Defaults to "buyer".
ownerName string True Merchant business name.
template string False Target format: "minimal", "fintech", or "glass".

Request Body (JSON)

{
  "apiKey": "fazzganzz",
  "amount": "Rp 750.000",
  "productName": "VIP Ticket Annual Pass",
  "buyerName": "Michael Scott",
  "buyerUsername": "mscott",
  "ownerName": "FazzPay Events",
  "ownerUsername": "fazzevents",
  "transactionId": "TXN-71937218319-FAZZ",
  "dateTime": "2026-07-01 19:45:00",
  "template": "glass"
}

Response Output

HTTP/1.1 200 OK
Content-Type: image/png
Content-Disposition: inline; filename="receipt.png"

Note: The payload will return raw image binary stream buffer for native rendering integration.

Error Responses

HTTP/1.1 401 Unauthorized

{
  "error": "Unauthorized. Invalid or missing API Key."
}
